I reviewed the uploaded `make local-dev` log. The app **starts successfully**, but the memory synthesis background job fails after login. The root cause is here: `MemoryService.synthesize_l3()` uses `asyncio.gather(...)` while sharing the **same SQLAlchemy `AsyncSession`** across concurrent DB operations. SQLAlchemy async sessions are not concurrency-safe, so this triggers:

```text
sqlalchemy.exc.InvalidRequestError:
This session is provisioning a new connection; concurrent operations are not permitted
```

Then the session cleanup also fails because the session is already in a bad state:

```text
sqlalchemy.exc.IllegalStateChangeError:
Method 'close()' can't be called here; method '_connection_for_bind()' is already in progress
```

## Exact failing area

The failure points to:

```text
backend/app/modules/memory/service.py line 223
traces, l2_docs, l3_docs = await asyncio.gather(...)
```

and:

```text
backend/app/modules/memory/repository.py line 140
return list((await db.execute(q)).scalars().all())
```

That means multiple repository methods are calling `db.execute(...)` at the same time on the same `AsyncSession`.

## Best fix

Do **not** run DB queries concurrently with the same session.

Change this pattern:

```python
traces, l2_docs, l3_docs = await asyncio.gather(
    self.repository.list_traces(db, user_id=user_id),
    self.repository.list_documents(db, user_id=user_id, level="l2"),
    self.repository.list_documents(db, user_id=user_id, level="l3"),
)
```

to this:

```python
traces = await self.repository.list_traces(db, user_id=user_id)
l2_docs = await self.repository.list_documents(db, user_id=user_id, level="l2")
l3_docs = await self.repository.list_documents(db, user_id=user_id, level="l3")
```

This is the safest fix.

## Alternative fix, only if you really need concurrency

Use **separate sessions per concurrent task**, not one shared session:

```python
async def fetch_traces(user_id):
    async with session_scope() as db:
        return await self.repository.list_traces(db, user_id=user_id)

async def fetch_l2_docs(user_id):
    async with session_scope() as db:
        return await self.repository.list_documents(db, user_id=user_id, level="l2")

async def fetch_l3_docs(user_id):
    async with session_scope() as db:
        return await self.repository.list_documents(db, user_id=user_id, level="l3")

traces, l2_docs, l3_docs = await asyncio.gather(
    fetch_traces(user_id),
    fetch_l2_docs(user_id),
    fetch_l3_docs(user_id),
)
```

But for a memory synthesis job, sequential DB reads are probably simpler and more reliable.

## Also fix the bad error log

This line is not useful:

```text
"memory_synthesize_failed ...", "exception":"NoneType: None"
```

It probably means the logger is calling `logger.error(..., exc_info=True)` outside the active exception block, or using `logger.error` instead of `logger.exception`.

Use:

```python
except Exception:
    logger.exception(
        "memory_synthesize_failed",
        extra={"user_id": str(user_id)}
    )
```

inside the `except` block.
